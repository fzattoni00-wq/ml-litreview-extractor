# ml-litreview-extractor

A Claude Code skill and multi-agent workflow for analysing and classifying research papers to support structured literature reviews. The workflow was developed and validated on a corpus of 100+ papers focused on neural-network-based estimation of lithium battery state of health (SOH), state of charge (SOC), and remaining useful life (RUL). It generates a spreadsheet that maps each paper against 35 machine-learning and application-oriented features, enabling faster comparison, synthesis, and gap identification.


## Key strenghts

- **Benchmarking:** glance comparison of networks architectures, data preparations and performance metrics 
- **Domain-agnostic:** scalable to different applied machine learning domains
- **Paywall:** running on local PDFs can bring to another level of detail your Zootero libraries and avoid basing literature reviews on only open access papers 



## How it works

```
PDFs ─▶ [1] preprocess.py  ─▶ work/<id>/{text.txt, tables.txt, page_*.png}
                                  │
                                  ▼
       [2] extract_workflow.js  (per PDF: extractor agent → adversarial verifier)
                                  │  → {count, rows} (schema-validated JSON)
                                  ▼
       [3] write_excel.py  ─▶ review_FILLED.xlsx  (35 columns; uncertain cells yellow)
```

Designed for **publication-grade accuracy**:


- **One agent per PDF + a second adversarial pass.** Isolation avoids
  cross-contamination between papers; the verifier *tries to refute* every
  critical field, catching plausible-but-wrong values a single read lets through.
- **Vision on result pages.** The preprocessor renders the result pages to PNG so
  the agents read metrics (RMSE, MAE, R², accuracy, F1, AUC…) from figures and
  image-tables, not just the text layer.
- **Uncertain cells are flagged** (highlighted yellow + listed in a
  `Fields to verify` column) so a human fact-check is fast and targeted.

Stages 1 and 3 are deterministic Python (no AI → no hallucination in prep or in
writing the spreadsheet). Stage 2 is the AI work, run as a schema-enforced
workflow so each agent returns a validated object, not free text.

## Classification mechanism 

Papers are classified according to a broad set of features, grouped into the following categories:

- **Paper metadata:** `PDF name`, `Paper title`, `Authors`, `Year`, `Publisher / venue`, `DOI / link`.
- **Application-oriented context:** `Target task`, `Target definition`, `Battery chemistry`, `Cell / module / pack level`.
- **Data and experimental setup:** `Dataset`, `Public dataset?`, `Experimental conditions`.
- **Inputs and feature preparation:** `Input data type`, `Data preparation`, `Feature extraction`.
- **Machine-learning model:** `Model category`, `NN architecture`, `Training strategy`, `Validation strategy`, `Baseline models`.
- **Results and reproducibility:** `Metrics reported`, `Main results`, `Generalisation tested?`, `Code/data available?`.
- **Synthesis:** `Advantages`, `Limitations`, `Relevance to my review`, `Notes`.
- **Analytical add-ons and quality control:** `Architecture (plain-language)`, `Input representation`, `Results (structured)`, `Results source + page`, `Extraction confidence`, `Fields to verify (manual)`.

Full per-column rules: [`references/column-schema.md`](references/column-schema.md).

## Visualisation

The classification spreadsheet is used to provide an overview into reviewed papers using the following plotted metrics:

- **Publications by year** (`01_publications_by_year.png`): yearly count of reviewed papers.
- **Publications by publisher venue** (`02_publications_by_publisher_venue.png`): detailed journal/conference venue count.
- **Publications by editorial publisher** (`02b_publications_by_editorial_publisher.png`): compact grouping by publisher such as Elsevier, MDPI, Springer Nature, IEEE, and IOP/ECS.
- **Battery chemistry** (`03_battery_chemistry.png`): normalized chemistry categories such as LFP, NMC/NCM, LCO, NCA, and unspecified Li-ion.
- **Dataset usage** (`04_dataset_used_amount.png`): count of common datasets and sources such as NASA PCoE, CALCE, Oxford, MIT-Stanford-TRI/Severson, and authors' own datasets.
- **Neural-network families** (`05_neural_networks_by_family.png`): architectures grouped into convolutional, recurrent, and attention-based families.
- **Author collaboration network** (`06_author_collaboration_network.png`): co-author graph highlighting recurring authors and collaboration clusters.

Plots are generated by [`post_processing/review_plotting.py`](post_processing/review_plotting.py) and written to `post_processing/plots/`.



## Requirements

- Python 3 with the dependencies listed in [`requirements.txt`](requirements.txt)
  (`pip install -r requirements.txt`)
- [Claude Code](https://claude.com/claude-code) with the multi-agent **Workflow**
  capability (the workflow runs in "ultracode" mode).

## Usage

1. Install as a Claude Code skill: copy this folder to
   `~/.claude/skills/ml-litreview-extractor/`.
2. Put your PDFs in a `pdfs/` directory and provide an `.xlsx` template (or let
   `write_excel.py` create the sheet).
3. Follow [`SKILL.md`](SKILL.md) (or paste [`PROMPT-ultracode.md`](PROMPT-ultracode.md)
   into a Claude Code session). The three stages run preprocess → workflow →
   write_excel and produce `review_FILLED.xlsx`.

## Note on the papers

This repository contains only the tooling. The /pdfs folder provides the list of paper DOIs used to develop and test the workflow ([`doi_list.txt`](pdfs/doi_list.txt)). PDFs are not included because they are copyrighted by their publishers, including IEEE, Elsevier, Springer, Taylor & Francis, Wiley, ACM, and AIP, and must not be redistributed. Please use your own legally obtained PDFs. All processing is performed locally on your machine.
